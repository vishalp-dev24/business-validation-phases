## NFX Network Effects Framework

### Network Effect Type Identification

#### 1. Direct Network Effects

**Physical** (hardware, phone):
- [ ] Product requires physical deployment
- [ ] Value increases with more compatible devices
- [ ] Example: Telephone networks, messaging apps

**Protocol** (standards, languages):
- [ ] Product is based on open/competing standard
- [ ] Standard adoption drives value
- [ ] Example: Ethernet, PDF, language apps

**Personal Utility** (communication, storage):
- [ ] Users use product for personal benefit
- [ ] Value increases with friends/contacts on platform
- [ ] Example: WhatsApp, Dropbox (sharing)

**Market Network** (professional communities):
- [ ] Combines marketplace + network + SAAS
- [ ] Critical mass needed on both sides
- [ ] Example: LinkedIn, Houzz

## Adjacent Market Expansion

### Testing Framework (Brian Balfour)

**Before expanding, validate:**

1. **Market Similarity Check**
   - Same core value proposition? Yes / No
   - Same acquisition channels work? Yes / No
   - Same buyer persona? Yes / No

2. **Landing Page Validation**
   - Create landing page for new market
   - Run ads for 2 weeks
   - Measure: Signups? Conversion rate?

3. **Beta Program**
   - 10-20 users from new market
   - Same onboarding flow
   - Measure: Activation? Retention?
